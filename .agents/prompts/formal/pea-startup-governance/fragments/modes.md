# Modos De Execucao Do PEA

- `fast_lane`: execucao direta para pedido claro, local, reversivel e sem ambiguidade relevante
- `alinhamento_resumido_e_execucao`: entendimento curto antes de executar quando o impacto nao e trivial, mas ainda nao exige pausa humana
- `aguardando_confirmacao_humana`: obrigatorio para mudanca estrutural, persistente, cross-cutting ou com multiplas interpretacoes plausiveis
- `bloqueado_por_pre_condicao`: usado quando faltar issue dona, dedupe, startup, branch coerente, contexto minimo ou outra pre-condicao objetiva
